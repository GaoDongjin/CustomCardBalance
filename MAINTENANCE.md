# Custom Card Balance：维护流程

## 1. 目标

本文档固化 `RevertCardsMod_Menu_Content` Excel 工作簿驱动的增量更新流程。每次更新分为“读取与确认”和“编码与部署”两个阶段。没有收到用户明确的“开始工作”指令时，只允许分析和输出对比表，不得修改代码或部署。

## 2. Excel 输入

当前工作簿：

`<WORKSPACE>\outputs\revertcardsmod-menu-content\RevertCardsMod_Menu_Content.xlsx`

固定文件名：

`RevertCardsMod_Menu_Content.xlsx`

如果今后文件移动位置或出现多个同名副本，必须先向用户确认应使用哪一个文件，不得自行猜测。

### 2.1 标签页

Excel 固定包含 6 个标签页：

1. 铁甲战士
2. 静默猎手
3. 故障机器人
4. 储君
5. 亡灵契约师
6. 其他

Excel 不创建“全部”标签页。Mod 设置面板中的“全部”由程序按照以上顺序合并生成。

### 2.2 表格列

每个标签页使用相同的 5 列：

| 列名 | 用途 |
| --- | --- |
| 卡牌名称 | 游戏中的中文卡牌名称 |
| 调整内容 | 设置面板中的调整摘要 |
| 升级前效果 | 计划应用的基础卡牌效果文本 |
| 升级后效果 | 计划应用的升级卡牌效果文本 |
| 开关 | 新增卡牌的默认开关状态 |

Excel 是维护输入和 UI 文案来源，但不能替代源码分析。实现卡牌时仍然必须检查目标游戏版本中的卡牌类、动态变量、升级逻辑、费用、关键词、打出效果、描述文本和悬浮信息。

工作簿文件名沿用历史名称 `RevertCardsMod_Menu_Content.xlsx`。该文件名属于维护输入，不是运行时 Mod ID。

## 3. 阶段一：读取与确认

当用户提出“更新 Mod”时，先执行以下步骤。

### 3.1 确认目标版本

1. 确认本次目标游戏版本。
2. 确认对应解包源码目录。
3. Ver0.106 系列默认参考：

`<STS2_SOURCE_DIR>`

如果用户要求适配新的游戏版本，必须先读取新版本解包源码。不得沿用 Ver0.106 的实现假设。

### 3.2 完整读取 Excel

读取 6 个标签页中的全部卡牌行，不得只读取最后新增的几行，也不得只读取用户口头提到的卡牌。

### 3.3 读取当前 Mod

至少读取：

- `ModConfiguration.cs` 中的 `CardIds` 和 `CardIds.All`
- `SettingsPanel.cs` 中的 `_entries`
- `CardPatches.cs` 中的 Harmony 补丁
- `CustomCardBalance.json` 中的 Mod 版本和最低游戏版本

### 3.4 分类差异

把 Excel 与当前 Mod 对比后，将卡牌分为：

| 分类 | 含义 | 默认处理 |
| --- | --- | --- |
| 新增候选卡牌 | Excel 中存在，但当前 Mod 尚未包含 | 进入源码对比表 |
| 已有卡牌且 Excel 文案变化 | 当前 Mod 已包含，但 Excel 内容与代码不同 | 单独列出差异，等待用户明确决定 |
| 已有卡牌且内容一致 | Excel 与代码一致 | 不修改 |
| 代码中存在但 Excel 缺失 | 当前 Mod 已包含，但 Excel 中没有 | 单独提示，不得自动删除 |

默认只处理新增候选卡牌。已有卡牌不得因为 Excel 文案变化而自动改动，除非用户明确要求同步调整。

### 3.5 检查游戏源码

对每张新增候选卡牌，读取目标游戏版本中的对应解包源码，至少核对：

- 卡牌所属角色和内部类名
- 稀有度和费用
- 升级前效果
- 升级后效果
- 动态变量
- 关键词，例如消耗、虚无、保留、固有
- `OnUpgrade`
- `OnPlay`
- 本地化描述
- 悬浮信息
- 是否与其他卡牌、能力、遗物或牌堆操作存在联动

如果 Excel 文案不足以确定运行逻辑，必须把疑点写入对比表，不得自行补全设计。

### 3.6 输出确认表

向用户输出新增候选卡牌对比表，至少包含：

| 列名 | 内容 |
| --- | --- |
| 所属分类 | 角色标签页 |
| 卡牌名称 | 中文名称 |
| 内部类名或 ID | 源码中的识别信息 |
| 当前游戏版本升级前效果 | 目标版本原版 |
| 当前游戏版本升级后效果 | 目标版本原版 |
| Excel 计划升级前效果 | 用户维护内容 |
| Excel 计划升级后效果 | 用户维护内容 |
| 调整内容摘要 | Excel 文案 |
| 预计实现位置 | 费用、变量、关键词、升级、打出效果、描述等 |
| 待确认问题 | 任何源码与 Excel 无法直接对齐的地方 |

已有卡牌的文案变化和 Excel 缺失项应放在单独的小节中提示。

输出对比表后停止工作，等待用户确认或修改 Excel。

## 4. 阶段二：编码与部署

只有在用户检查对比表，并明确下达“开始工作”指令后，才能执行本阶段。

### 4.1 重新读取最终输入

编码前再次读取完整 Excel，确认用户是否修改过内容。若出现未经确认的新卡牌或新的歧义，停止编码并重新输出差异。

### 4.2 修改代码

新增卡牌通常需要同步修改：

| 文件 | 维护内容 |
| --- | --- |
| `ModConfiguration.cs` | 新增稳定的卡牌 ID，加入 `CardIds.All`，默认开启 |
| `SettingsPanel.cs` | 新增分类、卡牌名称、调整摘要、升级前效果、升级后效果和开关 |
| `CardPatches.cs` | 根据目标源码增加最小范围 Harmony 补丁 |
| `CustomCardBalance.json` | 更新描述中的卡牌数量；需要时提高 Mod 版本和最低游戏版本 |

补丁原则：

- 开关开启时应用调整逻辑。
- 开关关闭时继续执行当前游戏版本的原版逻辑。
- 只修改已确认卡牌。
- 不顺手重构无关代码。
- 如果调整影响描述、关键词或悬浮信息，必须同步处理 UI 展示。

### 4.3 补充验证

根据新增卡牌的行为补充回归检查。已有固定检查：

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\verify-hotkey-entry.ps1
powershell -ExecutionPolicy Bypass -File .\tests\verify-settings-layout.ps1
powershell -ExecutionPolicy Bypass -File .\tests\verify-custom-card-balance-expansion.ps1
powershell -ExecutionPolicy Bypass -File .\tests\verify-custom-card-balance-rename-and-lazy-ui.ps1
dotnet build -c Release --no-restore
```

至少确认：

- Release 构建为 `0` 警告、`0` 错误。
- 初始化阶段只挂载空面板，不得提前构建完整表格。
- 面板仍然只能在主菜单通过 `F1` 打开。
- 设置菜单备用入口仍然存在。
- 新卡牌在正确标签页出现。
- “全部”页按照角色顺序合并展示。
- 新卡牌默认开关为开启。
- 关闭新卡牌开关后，游戏执行当前版本原版逻辑。
- 长文本没有破坏表格布局。

### 4.4 备份与部署

部署前确认游戏进程未运行。

备份目录格式：

`<BACKUP_ROOT>\CustomCardBalance-YYYYMMDD-HHMMSS`

至少备份：

- Steam `mods` 目录中的旧 `RevertCardsMod` 部署目录
- Steam `mods` 目录中的旧 `CustomCardBalance` 部署目录（如果存在）
- D 盘源码镜像中的被修改源码文件

部署目录：

`<STS2_GAME_DIR>\mods\CustomCardBalance`

部署内容：

- `CustomCardBalance.dll`
- `CustomCardBalance.json`

同时把更新后的源码和新增测试同步到：

`<SOURCE_MIRROR_DIR>`

### 4.5 部署核对

部署后核对：

- 工作副本 DLL 哈希
- D 盘源码镜像 Release DLL 哈希
- Steam `mods` 目录 DLL 哈希
- 关键源码文件哈希
- 新增测试文件哈希

三处 DLL 哈希必须一致。

## 5. 当前 Ver0.106 基线

当前已实现 21 张卡牌：

- 铁甲战士：主宰、跃跃欲试、被遗忘的仪式
- 静默猎手：杂技、触不可及、预判、速行者、谋杀、幽魂形态
- 故障机器人：电流相生、彩虹
- 储君：辉光、星位序列、虚空形态、封印王座
- 亡灵契约师：女妖之嚎、挽歌、降灵、预借时间、违逆
- 其他：生产制造

Excel 中出现但未写入当前代码的卡牌仍然只是候选项。它们必须经过“读取源码 → 输出对比表 → 用户确认 → 用户下达开始工作指令”的完整流程。

## 6. 每次更新的简版检查单

### 分析阶段

- [ ] 确认目标游戏版本和解包源码目录
- [ ] 读取 Excel 全部 6 个标签页
- [ ] 读取当前 Mod 卡牌清单与补丁
- [ ] 找出新增、已有变化、缺失三类差异
- [ ] 对新增候选逐张读取游戏源码
- [ ] 输出对比表
- [ ] 等待用户确认

### 实施阶段

- [ ] 收到明确的“开始工作”指令
- [ ] 重新读取最终 Excel
- [ ] 只实现已确认内容
- [ ] 补充回归检查
- [ ] 运行固定检查和 Release 构建
- [ ] 核对延迟创建 UI 和启动计时日志仍然存在
- [ ] 确认游戏未运行
- [ ] 备份旧文件
- [ ] 同步源码并部署 DLL 与清单
- [ ] 核对三处 DLL 哈希
- [ ] 汇报改动、验证结果和备份路径

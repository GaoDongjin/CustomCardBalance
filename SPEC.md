# Custom Card Balance：Ver0.106 规格

## 1. 文档定位

本文档固化 `Custom Card Balance` 在《杀戮尖塔 2》`Ver0.106` 系列中的当前基线。后续维护不得根据旧版记忆直接修改逻辑，必须以目标游戏版本的解包源码和本文档为准。

- 对外名称：`Custom Card Balance`
- 内部 ID：`CustomCardBalance`
- 作者：`Bruiser`
- 当前 Mod 版本：`2.2.0`
- 目标游戏版本：`Ver0.106`
- 当前清单中的最低兼容版本：`0.106.1`
- Ver0.106 解包源码：`<STS2_SOURCE_DIR>`
- Steam 部署目录：`<STS2_GAME_DIR>\mods\CustomCardBalance`
- 源码镜像：`<SOURCE_MIRROR_DIR>`

## 2. 核心行为

Mod 为每张纳入范围的卡牌提供独立开关：

- 开启：应用 Mod 中定义的调整效果。
- 关闭：完全保留玩家当前游戏版本中的原版效果。
- 默认状态：所有纳入 Mod 的卡牌均为开启。
- 保存后：游戏自动退出，玩家重启游戏后设置生效。

任何卡牌关闭开关后，都不得强行回调到某个固定旧版本。Harmony 补丁必须在开关关闭时让游戏继续执行当前版本的原版逻辑。

## 4. 设置面板

### 4.1 打开方式

- 仅在主菜单中允许按 `F1` 打开或关闭面板。
- 对局中不得通过 `F1` 调出面板。
- 游戏原生“设置”菜单中额外提供一行 `Custom Card Balance` 入口，按钮文字为 `打开设置`。
- Mod 不依赖 `BaseLib`。

### 4.2 标签页

顶部固定包含 7 个标签页：

1. 全部
2. 铁甲战士
3. 静默猎手
4. 故障机器人
5. 储君
6. 亡灵契约师
7. 其他

打开面板时默认显示“全部”。“全部”按照以上角色顺序合并展示卡牌。“其他”排在最后。

没有纳入卡牌的角色标签仍然可以点击，表格区域显示：

`这个角色暂时没有调整的卡牌`

### 4.3 表格

表格固定包含 5 列：

| 列名 | 用途 |
| --- | --- |
| 卡牌名称 | 中文卡牌名称 |
| 调整内容 | 简要说明 Mod 修改了什么 |
| 升级前效果 | 开启调整后的基础卡牌效果 |
| 升级后效果 | 开启调整后的升级卡牌效果 |
| 开关 | 单独启用或关闭该卡牌调整 |

已确认的布局参数：

- 效果列宽度：`450px`
- 卡牌数据行高度：`72px`
- 卡牌名称左侧内边距：`6px`
- 效果正文允许自动换行，不得裁切
- 开关列位于黑色表格区域最右侧，不得超出边框

### 4.4 底部按钮

- `恢复默认`
- `取消`
- `保存并退出`

顶部提示文字：

`保存后游戏会自动退出，重启后生效`

### 4.5 启动性能

- 进入主菜单时只挂载空面板，不得立即创建表格节点。
- 首次按 `F1` 或点击设置入口时，才构建完整设置 UI。
- 后续重复打开面板时复用已创建的 UI。
- 日志分别记录 Mod 初始化、主菜单空面板挂载、首次完整 UI 构建耗时。

## 5. 设置持久化

设置文件路径由 Godot 用户数据目录决定：

`<Godot User Data Dir>\CustomCardBalance\settings.json`

保存格式为 JSON。新增卡牌加入 Mod 时，默认开关必须为开启。加载旧设置文件时，仅接受当前代码中已登记的卡牌 ID；缺少的新卡牌使用默认开启状态。

首次加载时，如果新目录中不存在设置文件，但旧目录 `<Godot User Data Dir>\RevertCardsMod\settings.json` 存在，则复制旧设置文件到新目录。旧目录只作为迁移来源保留。

## 6. 代码职责

| 文件 | 职责 |
| --- | --- |
| `Plugin.cs` | 加载配置并注册 Harmony 补丁 |
| `ModConfiguration.cs` | 卡牌 ID、默认开关、JSON 读写、旧配置迁移 |
| `CardPatches.cs` | 卡牌效果、升级、费用、关键词、描述和悬浮信息补丁 |
| `SettingsPanel.cs` | 主菜单空面板、延迟创建 UI、`F1` 入口、设置菜单入口、标签页和表格 |
| `CustomCardBalance.json` | Mod 清单、版本和最低游戏版本 |

## 7. 验证基线

每次修改至少执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\tests\verify-hotkey-entry.ps1
powershell -ExecutionPolicy Bypass -File .\tests\verify-settings-layout.ps1
powershell -ExecutionPolicy Bypass -File .\tests\verify-custom-card-balance-expansion.ps1
powershell -ExecutionPolicy Bypass -File .\tests\verify-custom-card-balance-rename-and-lazy-ui.ps1
dotnet build -c Release --no-restore
```

构建通过后，部署前必须确认游戏进程未运行。部署完成后，必须核对工作副本、D 盘源码镜像和 Steam `mods` 目录中的 DLL 哈希一致。

## 8. 范围边界

- 本文档只描述 Ver0.106 系列 Mod。
- Excel 中存在的候选卡牌不等于已经纳入 Mod。
- 后续游戏版本不得静默复用 Ver0.106 的源码判断；必须重新读取对应版本解包源码并单独确认。
- 不得在没有用户确认的情况下新增、删除或修改卡牌补丁。
